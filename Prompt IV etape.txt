Je souhaite que tu remplisses ma base de données avec un nombre conséquent de données. Voici le MLD de ma base de données : 

Salle = (id_salle INT, nom_salle VARCHAR(50), adresse_salle VARCHAR(50), ville_salle VARCHAR(50), code_postal_salle INT, capacite_salle INT, horaire_salle VARCHAR(50));  id_salle INT PRIMARY KEY Secteur = (id_secteur INT, nom_secteur VARCHAR(50), #id_salle);  id_secteur INT PRIMARY KEY session_ouverture = (#id_salle, id_session INT, date_session INT);  PRIMARY KEY (id_salle, id_session) Prise = (id_prise INT, couleur_prise VARCHAR(50), type_prise VARCHAR(50), matiere_prise VARCHAR(50), quantite_prise INT, date_prise INT);  id_prise INT PRIMARY KEY Client = (id_client INT, nom_client VARCHAR(50), prenom_client VARCHAR(50), email_client VARCHAR(50));  id_client INT PRIMARY KEY abonnement = (id_abonnement VARCHAR(50), types_abonnement VARCHAR(50), date_debut INT, date_fin INT, #id_salle, #id_client);  id_abonnement VARCHAR(50) PRIMARY KEY, produit = (id_produit INT, categorie_produit VARCHAR(50), quantite_produit INT, prix_produit VARCHAR(50));  id_produit INT PRIMARY KEY Employe = (id_employe INT, nom_employe VARCHAR(50), prenom_employe VARCHAR(50), role_employe VARCHAR(50), #id_employe_1*);  id_employe INT PRIMARY KEY Bloc = (id_bloc INT, couleur_bloc VARCHAR(50), date_creation_bloc INT, date_retrait_bloc VARCHAR(50), #(#id_salle, id_session), #id_employe, #id_secteur);  id_bloc INT PRIMARY KEY composer = (#id_bloc, #id_prise);  PRIMARY KEY (id_bloc, id_prise) achat = (#id_client, #id_produit, date_achat INT, quantite INT);  PRIMARY KEY (id_client, id_produit, date_achat) 

Les clés primaires correspondent aux id, sauf si autre chose est précisé (quand c'est un attribut composé) les clés étrangères sont identifiées par les #, et ont le même nom que les clés primaires auxquelles elles font référence. Je veux que tu génères un minimum de 10 lignes par TABLE. Les clés étrangères doivent faire référence aux clés primaires existantes : donne les lignes en commençant par remplir les tables dans lesquelles il n'y a pas de clés étrangères, puis les tables dans lesquelles les clés étrangères font références à des clés primaires des tables déjà remplies.
